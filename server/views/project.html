Project

Researchers in the Life Sciences are faced with an increasing abundance of data in scientific repositories. Thus, handling and filtering of data becomes a more and more time-consuming and challenging task. Personalized applications are one solution and help to keep the focus on relevant topics. However, personalization requires to collect user information stored in so-called user profiles. They are often black boxes and less or no provenance information is provided where the entries come from. In addition, manual creation of user profiles usually puts a burden on the user. Therefore, in a first collaboration, we developed ScholarLens, a text mining pipeline that takes a scholar's publications and creates a semantic user profile containing relevant competences.

Semantic user profiles are more complex as keyword profiles are. Instead of plain keywords, we store a user's competences as triples and link the extracted competences to DBpedia entries by means of DBpedia Spotlight. Hence, proper visualization and explanation techniques are required to display what user information has been recorded and why. Provenance and transparency are key criteria in the development process of ScholarLensViz, a framework built on top of ScholarLens to visualize semantic user profiles.

Both, ScholarLens and ScholarLensViz were supported by the German Academic Exchange Service (DAAD)
